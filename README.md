# Neurobloom
Hello,

This is my second serious attempt at making a neural network library. The first library was good but not terribly efficient complexity wise and it didn't allow me to easily add these "monster" features I want to add. I have an idea to create a neural network library and language that allows users to do things most libraries won't let you. These "things" are what I call monster features. These monster features include but are not limited to, residual connections (weighted and non-weighted)(withing and external to a network), mult-network integrations, variable activation functions, cost analysis on any node, "husk networks" which just store node values from a network for "remembering" values, gaussian noise to limit weight information, and more.

I also want to make my own programming language to use this library too. Currently making a simple network in C++ involves something like ~50 lines of code (mostly function calls) which is really stupid. I feel like one should be able to create a network with no more than 10 lines of code for a simplistic network so I'm building a language (called Quack) to do this. I'm going to make it easy to use, easy to learn, and very light on the number of characters you need to type to write commands, kind of like assembly (funny how we went full circle there eh?).
